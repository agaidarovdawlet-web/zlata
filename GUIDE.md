# Гайд по пользованию

## Первый запуск

После установки откройте приложение `Т+`.

На первом запуске БД автоматически поднимается из `assets`, поэтому интернет не нужен.

## Главный экран

На главной странице показаны:

- число клиентов;
- количество активных счетов;
- число счетчиков;
- общий долг;
- показания за текущий месяц;
- счетчики, у которых поверка скоро истечет.

Ниже есть быстрые карточки с последними счетчиками и платежами.

## Поиск

В верхней строке поиска можно вводить:

- фамилию или имя клиента;
- номер лицевого счета;
- улицу или город;
- номер или тип счетчика.

Поиск обновляет списки сразу.

## Раздел `Счета`

Здесь отображаются лицевые счета.

Что видно в строке счета:

- номер лицевого счета;
- ФИО клиента;
- адрес;
- тип услуги;
- статус;
- баланс.

Если у счета есть долг, появляется кнопка `Оплатить`.

## Раздел `Счётчики`

Здесь список приборов учета.

Для каждого счетчика показаны:

- тип;
- серийный номер;
- лицевой счет;
- место установки;
- статус поверки;
- последнее показание.

Кнопка `Внести` открывает форму добавления показания.

Кнопка `История` показывает историю показаний выбранного счетчика.

## Внесение показания

В форме нужно указать:

- дату;
- значение показания;
- способ передачи.

После сохранения показание попадет в локальную БД и список обновится.

## Раздел `Оплаты`

В этом разделе показаны платежи по лицевым счетам.

Если у счета есть отрицательный баланс, оплату можно добавить из списка или через плавающую кнопку действия.

## Раздел `Клиенты`

Здесь список всех клиентов компании.

В строке клиента видны:

- ФИО;
- регион;
- телефон;
- email.

## Фильтры

На экране лицевых счетов есть фильтры:

- `Все`
- `Долг`
- `Активные`

На экране счетчиков есть фильтры:

- `Все`
- `Поверка скоро`
- `Просрочены`

## Что делать, если данные не обновились

Если вы добавили показание или оплату, но не видите изменения сразу:

1. Переключитесь на другой раздел.
2. Вернитесь обратно.
3. Используйте поиск заново.

## Для отчета

В пояснительной записке можно писать, что приложение:

- работает с локальной SQLite-базой;
- хранит данные о клиентах, счетах, счетчиках, показаниях и оплатах;
- работает офлайн;
- использует Material 3;
- подходит для сотрудников энергосбытовой компании.
